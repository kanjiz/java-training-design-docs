# RouteInputPrinter 詳細設計書

## 1. 機能要件

- ユーザーから出発地と到着地をキーボード入力で受け取る
- 入力された2つの地点を使用して往復メッセージを表示
- メッセージは入力された地点を結合して往復表現を作成

## 5. 入出力設計

### 5.1 入力仕様

1. 入力方法
   - システムはユーザーから2回の文字列入力を受け付ける
   - 各入力前に適切なプロンプトを表示
   - 入力された文字列は一時保存される

2. 入力値の制約
   - 任意の文字列を許容
   - 空文字列も有効な入力として扱う
   - 最大文字数の制限なし

※具体的な入力方法の実装については、各言語のガイドラインを参照してください。

### 5.2 出力仕様

1. プロンプトメッセージ
   - 出発地入力用：">出発地："
   - 到着地入力用：">到着地："

2. 往復メッセージ
   - 形式："{出発地}～{到着地}を往復します。"
   - {}内は入力された文字列で置換

3. 出力例：

   ```text
   >出発地：岡山
   >到着地：東京
   岡山～東京を往復します。
   ```
